Prise en main de Stata

Stata est le logiciel de référence pour l'analyse de données en économie. Ce cours part de zéro : on apprend l'interface avant de lancer les premières régressions.

Ce que couvre ce cours

Interface (fenêtres, éditeur de données, éditeur de commandes), import de données, commandes de régression de base, manipulation de données (generate, drop, keep).

Commande clé : regress

La commande regress y x1 x2 lance une régression linéaire de y sur x1 et x2, et affiche coefficients, écarts-types, tests t et R² directement.

Sujet type — inspiré d'un TP de Licence 2, Université de Kinshasa

On dispose d'une base de données `salaire.dta` avec les variables salaire, experience et diplome. Écrivez les commandes Stata pour : (1) charger la base, (2) régresser salaire sur experience et diplome, (3) afficher les statistiques descriptives de salaire.

Voir le corrigé

use salaire.dta, clear

regress salaire experience diplome

summarize salaire

Remarque : use ... , clear évite un message d'erreur si des données sont déjà chargées en mémoire. summarize peut s'écrire en abrégé sum.

Sujet proposé par TutoratÉco

Après une régression, vous voulez tester si le modèle souffre d'hétéroscédasticité. Quelle commande utilisez-vous, et que faites-vous si le test est significatif ?

Voir le corrigé

On utilise estat hettest juste après regress. Si le test est significatif (p < 0,05), l'hypothèse d'homoscédasticité est rejetée. Dans ce cas, on relance la régression avec des écarts-types robustes : regress salaire experience diplome, robust.
